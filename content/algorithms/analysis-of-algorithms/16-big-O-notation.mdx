**Big-O**, bir algoritma verminin input girişinin boyutuna bağlı olarak worst-case senaryado nasıl olduğunu gösteren bir matematiksel notasyon standardıdır. Bir algoritmanın verimini ölçmek için şunlara ihtiyacımız olur:

- **Time Complexity:** Tamamen çalışmak için ne kadar zaman alır?
- [Space Complexity](https://cs-space.vercel.app/algorithms/analysis-of-algorithms/space-complexity): Bu süreçte ne kadar ekstra alan gerekir?

Big-O notasyonu bir algoritmanın worst-case senaryoda giriş boyutu büyüdükçe ne kadar zaman ya da alan gerektirdiğini göstermek için üst sınırı (upper bound) yakalar. Bu genellikle şu şekilde yazılır:

    f(n) = O(inputSize)

## Complexity Nasıl Hesaplanır?

Time complexity iki faktör dikkate alınarak belirlenir. Giriş boyutu (size) ve algoritmanın çözümü. Aşağıda complexity’i çözmek için genel yol açıklanlanmıştır:

- Koddaki bütün temel işlemleri listele
- Her bir işlemin kaç kez yürütüldüğünü say
- n cinsinden bir denklem elde etmek için tüm sayıları toplayın

:::note Örnek
Aşadağıdaki kodu inceleyelim ve eğer giriş boyutu n’e eşitse complexity’i nasıl hesaplayacağımıza bakalım:
:::

```jsx
#include <iostream>
using namespace std;

int main() {
  int sum = 0;
  for (int i=0;i<5;i++){
    sum = sum+i;
  }
  cout << "Sum = " << sum;
  return 0;
}
```

| İşlemler                | Yürütme Sayısı |
| ----------------------- | -------------- |
| int sum = 0             | 1              |
| for (int i=0;i<5;i++)   | 6              |
| cout << "Sum = " << sum | 1              |
| return 0                | 1              |

**Hesaplama**

- 1+6+5+1+1

Bu notasyonu giriş boyutu (n) cinsinden genelleştirmek şu ifadeyi oluşturur:

▶️ 1+(n+1) + n + 1+1

Yukarıdaki ifadeden sonra basitleşitrmek için sonuç olarak time complexity şu şekilde olacaktır:

▶️ 2n +4

### Bir Algrotimanın Big-O Notasyon’unu Nasıl Tahmin Ederiz?

Big-O notasyonu bulmak için şu adımları dikkate alırız:

- n’in önündeki sabitleri at
- Düşük dereceli terimleri görmezden gel (2n + 4n => 4n)

Yukarıdaki iki adımdan sonra big-o notasyonu hesaplayabiliriz:

    => 2n+4 => n+4 => n  => O(n)
